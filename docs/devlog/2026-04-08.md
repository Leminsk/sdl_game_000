# 2026 April 08 - Not so random colors
Sometimes I don't actually care which color I'm assigned to as the player. But past me never got around to implementing a random color picker, which means that so far I was forced to pick the color every time I wanted to test something in the main game scene. In the past this was so bad that attempting to play a map with the option "Random" selected would cause the game to crash!  

Up until now this was the result:  
<img src="/sdl_game_000/assets/images/2026-04-08/not_random_colors.gif" alt="PINK belongs to the player" style="max-width: 100%;">  
The spawn pixel has a pink color (COLORS_SPAWN=#FFAEC9) and so I use that as the value for "random color" as well. Since the player *technically* picked that color, now everything colored pink is now under the player's control :)  

That's not what I intended. Also the drones, or roombas as a friend of mine has nicknamed them, got that weird shade due to some color modulation going on.  

I thought about doing something clever and maybe doing some sampling of the colors but just a straightforward `std::vector` manipulation with a `randomInt()` function which I had already implemented is just what I needed. So now I have random color picker finally:  
<img src="/sdl_game_000/assets/images/2026-04-08/yay_random_colors.gif" alt="Now with random colors!" style="max-width: 100%;">
