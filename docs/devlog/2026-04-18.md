# 2026 April 18 - A minimap
At times when debugging I wish I knew more precisely where on the map I was looking at. In most, if not all RTS games there is some way to interact with a minimap in the corner of the screen, but I think it's a bit too soon to implement a "proper" minimap HUD, but at least knowing where the camera is should be a good start.  

For this initial version, I've repurposed my `MapThumbnailComponent.hpp`, which I use in the map selection menu below:
<img src="/sdl_game_000/assets/images/2026-04-18/first_minimap.png" alt="Just a way to display the BMP files" style="max-width: 100%;">  

Now when the map is loaded, I create a new instance of a `MapThumbnailComponent` just for the minimap, and since I already have the map data pre-loaded from the previous scene, I can just pass a reference to the same `std::vector<std::vector<SDL_Color>> map_pixels` that this scene already used to load everything else (no need to reload the BMP). Then I set it to be positioned in the bottom right corner of the screen:  
<img src="/sdl_game_000/assets/images/2026-04-18/first_minimap.gif" alt="Basic camera tracking" style="max-width: 100%;">  

I also clamp the values of the 4 camera rectangle corners so that it doesn't draw *outside* the minimap, which looks funny, but is janky and annoying.
