# 2026 April 18 - A minimap
At times when debugging I wish I knew more precisely where on the map I was looking at. In most, if not all, RTS games there is some way to interact with a minimap in the corner of the screen, but I think it's a bit too soon to implement a "proper" minimap HUD, but at least knowing where the camera is should be a good start.  

For this initial version, I've repurposed my `MapThumbnailComponent.hpp`, which I use in the map selection menu below:
<img src="/sdl_game_000/assets/images/2026-04-18/map_selection.png" alt="Just a way to display the BMP files" style="max-width: 100%;">  



The minimap under the hood really is just the `MapThumbnailComponent` without the title and dimensions legends and with an added logic to calculate on every frame what are the corners of the camera crop, the pink rectangle (technically a frustrum, but we're not in 3D).  

I already had implemented a `convertScreenToWorld()` previously which takes a `Vector2D`, basically a pair \[x,y\] class with some helper methods and functions, and converts that screen point position into its game world equivalent based on *where* the camera is pointing. This is the main part that deals with the calculations of the 4 camera rectangle corners:  
```cpp
/*
get the camera world position
scale it by the minimap proportions
*/
std::vector<Vector2D> screen_corners = std::vector<Vector2D>(4);
screen_corners[0] = convertScreenToWorld({                                   0.0f,                                    0.0f });
screen_corners[1] = convertScreenToWorld({ static_cast<float>(Game::SCREEN_WIDTH),                                    0.0f });
screen_corners[2] = convertScreenToWorld({ static_cast<float>(Game::SCREEN_WIDTH), static_cast<float>(Game::SCREEN_HEIGHT) });
screen_corners[3] = convertScreenToWorld({                                   0.0f, static_cast<float>(Game::SCREEN_HEIGHT) });
SDL_FPoint p;
for(int i=0; i<4; ++i) {
    screen_corners[i].x = (screen_corners[i].x * this->minimap_proportion_width) + this->origin_x;
    screen_corners[i].y = (screen_corners[i].y * this->minimap_proportion_height) + this->origin_y;
    p.x = std::clamp(screen_corners[i].x, this->map_rect.x, this->map_rect.x+this->map_rect.w);
    p.y = std::clamp(screen_corners[i].y, this->map_rect.y, this->map_rect.y+this->map_rect.h);
    this->camera_points[i] = p;
}
this->camera_points[4] = this->camera_points[0];
TextureManager::DrawWireframe(this->camera_points, 5, COLORS_SPAWN);
```  
The camera corners are by definition the corners of the window that the game is running on. First we treat those corners as `Vector2D` and find out where in the game world they are. Now that we know world wise what their points are, we need to convert them to the scale of the minimap (the multiplication part) which is the ratio of the size of the minimap in pixels on the screen (set on the creation of the `MapThumbnailComponent` object) divided by the size of the entire map in world units.  

Because the minimap does not need to use the same proportion as the original map, then we must use a separate proportion ratio for width and height.  

I also need to translate the points by the origin point of the `MapThumbnailComponent`, which is where on the screen the objects is rendered. That way the corners are now relative to the top left corner of the minimap. Finally I clamp the four points to limit the rendering of the rectangle so that it doesn't draw *outside* the minimap, which looked funny when I was testing it, but it's too janky and annoying.  

Don't worry about that `DrawWireframe()`, it's just a helper function to call `SDL_RenderDrawLinesF()` which takes the converted points and speedly draws them every frame. I need to pass 5 points to it because if the last point happens to be equal to the first one, it will draw a closed circuit (at least in this case).

Now when the map is loaded, I create a new instance of a `MapThumbnailComponent` just for the minimap, and since I already have the map data pre-loaded from the previous scene, I can just pass a reference to the same `std::vector<std::vector<SDL_Color>> map_pixels` that this scene already used to load everything else (no need to reload the BMP). Then I set it to be positioned in the bottom right corner of the screen:
